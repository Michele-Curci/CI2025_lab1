# CI2025_lab1

Repository for lab 1 of the course Computational Intelligence

**Problem description:**

This is a multi-dimensional multi-knapsack problem, each item has a value and a weight vector containing the weight for each dimension, each knapsack has a capacity vector containing the capacity for each dimension.

The goal is to assign items to knapsacks with the objective of maximizing the total value subject to the capacity constraint.

**Solution representation:**

Each solution is represented as a vector of integers of length NUM_ITEMS

solution ∈ {-1, 0, 1, ..., NUM_KNAPSACKS-1}

Where -1 means that the item i is not assigned to any knapsack, otherwise the value corresponds the the index of the knapsack to which it is assigned. This encoding allows for a inherent solution to the problem of assigning an item to multiple knapsacks.

**Constraint handling:**

The function check_constraints() computes the total used capacity per knapsack and dimension and checks feasibility. If the solution violates any constraint, the repair_solution() function iteratively removes items with the lowest value-to-weight ratio until all capacities are respected.

The fitness function is defined as the total value if the solution is feasible, as the total value minus a penalty otherwise.

**Solution:**

To address this problem I implemented a simulated annealing (SA) algorithm. SA starts from a single solution and iteratively explores its neighborhood by mutating a few items. New solutions are accepted if they improve fitness or probabilistically according to: 

P(accept) = e^(**Δ**f/T), with T being the current temperature.

The temperature decreases according to a cooling schedule (cooling_rate), allowing gradual convergence.

Following I'll present two other algorithms I implemented, they were used only as tests and may not follow traditional implementations, technical concepts derive from previous personal studies.

**Genetic algorithm (GA):**

GA maintains a population of solutions that evolve over generations using:

- Tournament selection,
- Uniform crossover,
- Random mutation (assigning new knapsacks to some items),
- Elitism (preserving the best solution).

Each generation improves the population's overall quality, tracked through best and average fitness plots. GA provides good exploration but may require careful tuning of mutation and crossover probabilities.

**Ant Colony Optimization (ACO):**

ACO constructs probabilistic solutions based on pheromone trails and a heuristic desirability funtion:

P_ik ∝ (τ_ik)^α ⋅ (**η**_ik)^β

where:

- τ_ik is the pheromone level for assigning item i to knapsack k,
- **η**_ik is the heuristic value,
- α, β control the influence of pheromone and heuristic.

Pheromone trails are updated with evaporation and reinforcement based on fitness. ACO balances exploration and exploitation but can suffer from early convergence if evaporation is too low.

**Conclusion:**

As we can see al three algorithms manage to find feasible solutions, ACO excels in smaller problems due to its constructive nature, GA balances exploration and exploitation and is ideal for medium-sized search spaces, SA is scalable to large instances with limited computational overhead.
